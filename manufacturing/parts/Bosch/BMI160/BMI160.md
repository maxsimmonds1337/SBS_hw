# BMI160 — 6-Axis IMU (Accel + Gyro)

## Key Specs

| Parameter | Value |
|-----------|-------|
| Axes | 3-axis accelerometer + 3-axis gyroscope |
| Accel range | ±2g to ±16g (software selectable) |
| Gyro range | ±125°/s to ±2000°/s |
| Interface | SPI or I2C (CSB pin selects) |
| Supply (VDDIO) | 1.8V to 3.6V |
| Supply (VDD) | 1.71V to 3.6V |
| Current (normal) | 925 µA (accel + gyro active) |
| Package | LGA-14 (3.0 × 2.5 mm) |

## I2C Interface Configuration

With CSB = VCC (+3.3V), the BMI160 operates in I2C mode.

```
I2C address:
  SDO pin (pin 11) = OSDO, tied to GND → I2C address = 0x68
  (if SDO/OSDO tied HIGH → address = 0x69)
```

The I2C bus (SCL=PA15, SDA=PB2 in VESC/SBS) uses software I2C in VESC firmware —
PA15 and PB2 don't have hardware I2C alternate functions on STM32F405.

## I2C Pull-Up Resistors

Check that 2.2kΩ pull-up resistors are present on the SDA and SCL lines in the
BMI160.kicad_sch sub-sheet (R5 and R38 in the VESC 6 MK5 reference). These must
be present for I2C to function.

From the VESC 6 MK5 BMI160 sub-sheet: R5 (SCL pull-up, 2.2kΩ to VCC), R38 (SDA pull-up, 2.2kΩ to VCC) — verify these exist in BMI160.kicad_sch.

## Decoupling

VESC 6 MK5 uses C50 + C51 = 2× 2.2µF on VDDIO. Check BMI160.kicad_sch has:
- 100nF + 2.2µF (or 2× 2.2µF) on VDD/VDDIO

## VESC Firmware Configuration

In VESC Tool → App Settings → IMU:
- IMU type: BMI160
- I2C address: 0x68 (or 0x69 depending on SDO/OSDO tie)
- The Balance App (Onewheel mode) requires this IMU for pitch angle measurement

VESC firmware reads BMI160 via software I2C on PA15 (SCL) and PB2 (SDA).
These pins are configured as GPIO in the firmware, not hardware I2C.

## Notes

- Auxiliary interface pins (ASDx, ASCx, OCSB, OSDO) are for connecting an external
  magnetometer (e.g., BMM150) via the BMI160's internal I2C master. Leave unconnected
  if not using compass functionality (VESC Balance App does not require a magnetometer).
- The VESC 6 MK5 BMI160 sub-sheet also has 2.2µF caps (C50/C51) on the auxiliary
  pins — these are not strictly necessary but provide stability. Optional to add.
